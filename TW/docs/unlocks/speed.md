# 速度升級
執行速度現在加倍了。問題是無人機現在採收的速度比牧草生長還快，導致完全採收不到任何東西。為了解決這個問題，現在解鎖 [if](docs/scripting/if.md) 分支和 [can_harvest](functions/can_harvest) 函式。

## 在採收前檢查
到目前為止，我們只有 `True` 和 `False` 作為條件，對 `if` 而言這當然不是很有用。

新函式 `can_harvest()` 提供了更好的條件。`can_harvest()` 在無人機下方的植物可以被採收時回傳 `True`，否則回傳 `False`。

```
if can_harvest():
    # 執行某些操作
```

之所以能像這樣將這個函式當作條件，是因為它會回傳布林值。

回傳值本質上是指在函式執行完畢後，函式呼叫的表達式的計算結果等於其回傳的值。

當上述程式執行時會發生的事：
- if 被執行
- 呼叫 `can_harvest()`
- `can_harvest()` 執行它的操作
- `can_harvest()` 回傳 `True` 或 `False`
- 現在該陳述式變為 `if True:` 或 `if False:`
- 只有在可以採收時，才會執行程式碼區塊

現在我們可以使用 `if` 來防止無人機過早採收。
