# 偵錯
有時候程式碼就是無法正常運行，而你需要找出原因。這裡有幾個工具可以幫助你。

第一個是逐步執行程式。
你可以使用「執行」按鈕旁的按鈕或設定斷點進入逐步執行模式。

點選程式碼左側的斷點面板可新增斷點。
![](Breakpoints227)
當程式碼執行到斷點所在的行時，就會自動切換到逐步模式。

將滑鼠移到變數上時，會顯示它目前的值。

`print()` 函式也非常有用。無人機會將傳遞給它的任何值直接書寫在空中。

範例：

印出 "0.24"。

```
print(0.24)
```

印出 "True" 或 "False"。

```
print(can_harvest())
```

印出目前位置。
```
print(get_pos_x(), get_pos_y())
```

print 函式將數值直接印出到空中和[輸出](docs/output.md)頁面。

如果你想印出大量的數值，在空中書寫有時可能會有點慢。
在這種情況下，可以使用 `quick_print()` 函式，它只會印出到輸出視窗。

輸出視窗也會記錄警告和錯誤，所以如果操作未按預期運行，不妨查看一下輸出視窗。

當執行停止時，輸出也會被寫入遊戲資料夾中的 [output.txt](persistent_data_path/output.txt) 檔案中。
