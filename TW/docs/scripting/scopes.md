# 命名作用域
作用域決定哪些變數可以從何處存取。作用域基本上是名稱到值的對應。
它們的運作方式與 Python 大致相同。

有一個全域作用域，每個函式有自己的區域作用域。
當你定義一個變數時，它會被加入到目前的作用域。
函式定義外的任何東西都視為全域作用域的一部分。

`x = 1`
將值 `1` 指派給全域作用域中的名稱 `x`。

這個 `def` 陳述式會將一個函式指派給全域作用域中的名稱 `f`。

```
def f():
    `在 `f` 的區域作用域中，將值 `1` 指派給名稱 `y`。`
    y = 1

    `在 `f` 的區域作用域中，將一個函式指派給名稱 `g`。`
    def g():
        pass
```

`f()`
從全域作用域取出儲存在 `f` 的函式並呼叫它。

`print(y)`
此位於全域作用域的 print 陳述式會拋出錯誤，因為 `y` 從未在全域作用域宣告，因此無法在此讀取。
它只存在於 `f` 的區域作用域中。

## global 關鍵字
預設情況下，函式內的所有變數都會綁定到區域作用域，即使全域作用域存在同名變數也一樣。

```
x = 0

def f():
    x = 1
f()
print(x)
```

這段程式會印出 `0`，因為 `f` 內的區域變數 `x` 與全域變數 `x` 不是同一個變數，所以全域變數 `x` 保持不變。這一點很重要，否則函數呼叫可能會意外覆寫剛好與該函式區域變數同名的全域變數。

如果想寫入全域變數，必須明確使用 `global` 關鍵字。

```
x = 0

def f():
    global x
    x = 1
f()
print(x)
```

在這個範例中，`global x` 會將 `x` 綁定到上方定義的全域變數 `x`。此時印出的會是 `1`。
注意，修改全域變數通常是導致程式碼混亂的第一步，在這種混亂的程式碼中，程式的每個部分都會影響程式的其他部分，因此請勿過度使用。

## 迴圈和分支
迴圈和分支不會建立自己的作用域，因此在其內宣告的任何東西仍可在外部使用。

```
for i in range(3):
    pass
print(i)
```

這段程式會印出 `2`，因為 `for` 迴圈最後一次迭代將 `2` 指派給 `i`。
